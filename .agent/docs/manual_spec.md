# 使い方マニュアル画面仕様書

## 1. 概要
本システムの操作方法をユーザー（管理者・生徒）が参照できる「使い方マニュアル画面」を新規に作成する。
ユーザーが操作に迷った際に、自己解決できる環境を提供することを目的とする。

## 2. 画面構成案

### 2.1. 画面レイアウト
- **ヘッダー**: 既存の共通ヘッダーを使用（ホームに戻るリンクなど）。
- **左サイドバー（PC時） / 上部メニュー（SP時）**: 目次・カデゴリ一覧。
- **メインエリア**: 選択されたマニュアルの詳細内容を表示。

### 2.2. マニュアルのアクセス方法
- **管理者**: 管理画面のヘッダーまたはサイドメニューに「マニュアル」リンクを配置。
- **生徒**: 生徒用ダッシュボードのヘッダーまたはメニューに「マニュアル」リンクを配置。
- **ログイン画面**: ログイン方法が不明なユーザー向けに、ログイン画面下部にも「ヘルプ / マニュアル」リンクを配置することを検討。

## 3. 機能要件

### 3.1. マニュアル表示機能
- カテゴリごとにマニュアルを分類して表示する。
- **管理者向けカテゴリ例**:
    - 生徒の登録・管理方法
    - 進捗状況の確認・更新方法
    - アカウント設定
- **生徒向けカテゴリ例**:
    - ログイン・ログアウト方法
    - 自分の進捗確認方法
    - プロフィールの変更

### 3.2. 目次・ナビゲーション
- 左サイドバーにカテゴリと記事タイトルをツリー状、またはリスト状に表示。
- クリックすると右側のメインエリアの該当箇所にスクロール、またはページ遷移する。

### 3.3. レスポンシブ対応
- スマートフォン表示時は、目次をアコーディオンメニューまたはドロワーメニューに格納し、記事本文を読みやすく表示する。

### 3.4. (オプション) 検索機能
- キーワードでマニュアル内のテキストを検索し、該当する記事を表示する機能（初期フェーズではブラウザ検索で代用可とするが、将来的には実装推奨）。

## 4. 技術スタック
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS (既存デザインに合わせて調整)
- **Content**:
    - フェーズ1: ハードコード、またはJSON/Markdownファイルから読み込み。
    - フェーズ2: CMS連携（必要であれば）。今回は静的コンテンツとして実装を想定。
